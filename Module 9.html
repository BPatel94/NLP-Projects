Exercise 1 Grammar for Subject-Verb Agreement This grammar correctly parses: "I am happy." "She is happy." And rejects: "You is happy." "They am happy."

Grammar Rules: S -> NP_1SG VP_1SG
S -> NP_3SG VP_3SG
S -> NP_PL VP_PL
NP_1SG -> 'I'
NP_3SG -> 'She' | 'He' | 'It'
NP_PL -> 'You' | 'They'
VP_1SG -> 'am' ADJ
VP_3SG -> 'is' ADJ
VP_PL -> 'are' ADJ
ADJ -> 'happy'

import nltk
from nltk import CFG
from nltk.parse import ChartParser

# Define grammar
grammar = CFG.fromstring("""
S -> NP_1SG VP_1SG
S -> NP_3SG VP_3SG
S -> NP_PL VP_PL

NP_1SG -> 'I'
NP_3SG -> 'She' | 'He' | 'It'
NP_PL -> 'You' | 'They'

VP_1SG -> 'am' ADJ
VP_3SG -> 'is' ADJ
VP_PL -> 'are' ADJ

ADJ -> 'happy'
""")

parser = ChartParser(grammar)

def test_sentence(sentence):
    words = sentence.split()
    try:
        trees = list(parser.parse(words))
        if trees:
            print(f"✔ Accepted: {sentence}")
            for tree in trees:
                print(tree)
        else:
            print(f"✘ Rejected: {sentence}")
    except:
        print(f"✘ Rejected: {sentence}")

# Test cases
test_sentence("I am happy")
test_sentence("She is happy")
test_sentence("You is happy")
test_sentence("They am happy")

✔ Accepted: I am happy
(S (NP_1SG I) (VP_1SG am (ADJ happy)))
✔ Accepted: She is happy
(S (NP_3SG She) (VP_3SG is (ADJ happy)))
✘ Rejected: You is happy
✘ Rejected: They am happy

Results and Explanation The grammar correctly enforces subject-verb agreement based on person and number. Accepted Sentences: "I am happy." - correctly uses "am" with first-person singular. "She is happy." - correctly uses "is" with third-person singular. Rejected Sentences: "You is happy." - incorrect because "You" requires "are", not "is". "They am happy." - incorrect because "They" requires "are", not "am". Conclusion: The grammar successfully distinguishes between first-person singular, third-person singular, and plural subjects, ensuring that only grammatically correct sentences are parsed.

Exercise 2 Subsumption in Feature Structures A feature structure f1 subsumes f2 if: Every feature in f1 exists in f2 The values for those features are the same (or recursively subsume) In simple terms: f1 is more general, and f2 is more specific.

In [4]:
def subsumes(f1, f2):
    """
    Returns True if f1 subsumes f2
    """
    for key in f1:
        # Feature must exist in f2
        if key not in f2:
            return False
        
        # If value is a dictionary → recursive check
        if isinstance(f1[key], dict) and isinstance(f2[key], dict):
            if not subsumes(f1[key], f2[key]):
                return False
        else:
            # Values must match
            if f1[key] != f2[key]:
                return False
    
    return True

# TRUE case (f1 is more general)
f1 = {

}

f2 = {

    "NUM": "SG"
}

print("Test 1 (should be True):", subsumes(f1, f2))

f3 = {
    "CAT": "VP",
    "TENSE": {
        "TYPE": "PRESENT"
    }
}

f4 = {
    "CAT": "VP",
    "TENSE": {
        "TYPE": "PRESENT",
        "ASPECT": "PROG"
    }
}

print("Test 2 (should be True):", subsumes(f3, f4))


# FALSE case (value mismatch)
f5 = {
    "CAT": "NP",
    "NUM": "SG"
}

f6 = {
    "CAT": "NP",
    "NUM": "PL"
}

print("Test 3 (should be False):", subsumes(f5, f6))


# FALSE case (missing feature)
f7 = {
    "CAT": "VP",
    "TENSE": "PAST"
}

f8 = {
    "CAT": "VP"
}

print("Test 4 (should be False):", subsumes(f7, f8))

Test 2 (should be True): True
Test 3 (should be False): False

The subsumes() function checks whether all features in f1 are present in f2 with matching values. If f1 has fewer or more general features - True If any feature is missing or mismatched - False The function also handles nested feature structures using recursion.

Exercise 3 Subcategorization Grammar with Features This grammar distinguishes between transitive and intransitive verbs using the SUBCAT feature.

Rules: S -> NP[CASE=nom, AGR=?a] VP[AGR=?a] VP[SUBCAT=intrans, AGR=?a] -> IV[SUBCAT=intrans, AGR=?a] VP[SUBCAT=trans, AGR=?a] -> TV[SUBCAT=trans, AGR=?a, OBJCASE=?c] NP[CASE=?c] Key Features: SUBCAT: distinguishes transitive vs intransitive verbs AGR: ensures subject-verb agreement OBJCASE: ensures correct object case (acc/dat)

import nltk
from nltk import grammar, parse

fcfg_grammar = nltk.grammar.FeatureGrammar.fromstring("""

S -> NP[CASE=nom, AGR=?a] VP[AGR=?a]

VP[SUBCAT=intrans, AGR=?a] -> IV[SUBCAT=intrans, AGR=?a]
VP[SUBCAT=trans, AGR=?a] -> TV[SUBCAT=trans, AGR=?a, OBJCASE=?c] NP[CASE=?c]

# Noun Phrases
NP[CASE=nom, AGR=[NUM=sg, PER=1]] -> 'ich'
NP[CASE=nom, AGR=[NUM=sg, PER=3]] -> 'er'

NP[CASE=acc] -> 'dich'
NP[CASE=dat] -> 'dir'

# Intransitive Verbs
IV[SUBCAT=intrans, AGR=[NUM=sg, PER=1]] -> 'komme'
IV[SUBCAT=intrans, AGR=[NUM=sg, PER=3]] -> 'kommt'

# Transitive Verbs (accusative)
TV[SUBCAT=trans, OBJCASE=acc, AGR=[NUM=sg, PER=1]] -> 'sehe'
TV[SUBCAT=trans, OBJCASE=acc, AGR=[NUM=sg, PER=3]] -> 'sieht'

# Transitive Verbs (dative)
TV[SUBCAT=trans, OBJCASE=dat, AGR=[NUM=sg, PER=1]] -> 'folge'
TV[SUBCAT=trans, OBJCASE=dat, AGR=[NUM=sg, PER=3]] -> 'folgt'

""")

parser = nltk.parse.FeatureChartParser(fcfg_grammar)
